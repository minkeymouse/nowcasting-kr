# Block Dynamic Factor Model (Block DFM) 이해

## 1. 글로벌 팩터 vs 로컬 팩터

### 글로벌 팩터 (Global Factor)
- **벡터 형태**: 글로벌 팩터는 벡터입니다 (스칼라가 아님)
- **독립적 추정**: 로컬 팩터들을 concat한 것이 **아닙니다**
- **역할**: 모든 시계열에 공통적으로 영향을 미치는 전체 경제의 공통 변동을 포착
- **구조**: 첫 번째 블록(Block_Global)에 해당하며, 모든 시계열이 이 블록에 로드됨

### 로컬 팩터 (Local Factors)
- **블록별 독립**: 각 블록(예: Consumption, Investment, External)마다 별도의 로컬 팩터
- **역할**: 해당 블록 내 시계열들의 공통 변동을 설명
- **구조**: Block_Consumption, Block_Invest, Block_Extern 등

### 계층형 구조
```
전체 시계열
├── 글로벌 팩터 (모든 시계열에 영향)
└── 로컬 팩터들
    ├── Consumption 블록 팩터
    ├── Investment 블록 팩터
    └── External 블록 팩터
```

## 2. Block DFM의 요인 회전 (Factor Rotation)

### 요인 회전을 하지 않음
- Block DFM은 **구조적 제약(structural constraints)**을 사용
- 블록 구조가 이미 경제적 의미를 부여함
- 따라서 요인 회전(factor rotation)이 필요 없음
- 각 팩터는 블록 구조에 의해 자동으로 해석 가능

### 일반 DFM과의 차이
- **일반 DFM**: 요인 회전(예: Varimax)을 통해 해석 가능한 구조 찾기
- **Block DFM**: 블록 구조가 이미 해석을 제공하므로 회전 불필요

## 3. 요인간 공분산과 독립 가정

### 공분산 구조
- **Q 행렬**: 팩터 간 공분산 행렬
- **블록 대각 구조**: 코드에서 `block_diag` 사용
- **독립 가정**: 각 블록의 팩터들은 서로 독립적으로 추정됨
  - 글로벌 팩터와 로컬 팩터 간의 공분산은 0 (또는 매우 작음)
  - 로컬 팩터들 간의 공분산도 0

### 실제 구현
```python
# 각 블록마다 독립적으로 공분산 추정
Q_block = np.cov(e_clean.T)  # 블록 내 공분산
# 전체 Q는 블록 대각 행렬로 구성
Q = block_diag(Q_global, Q_consumption, Q_invest, Q_extern)
```

## 4. 분산-공분산 분해

### 원래 값의 분해
```
X_t = C * F_t + e_t
```
- **X_t**: 관측된 시계열 (N x 1)
- **C**: 로딩 행렬 (N x m)
- **F_t**: 팩터 벡터 (m x 1)
- **e_t**: 특이 충격 (idiosyncratic shocks)

### 분산-공분산 분해
```
Var(X_t) = C * Var(F_t) * C' + Var(e_t)
         = C * Q * C' + R
```
- **Q**: 팩터 간 공분산 행렬 (m x m)
- **R**: 특이 충격의 분산 행렬 (N x N, 대각 행렬)

### 고유분해(Eigendecomposition) 사용
코드에서 확인:
```python
# 각 블록마다 residual covariance의 eigendecomposition
eigenvals, eigenvecs = np.linalg.eigh(cov_res)
# 가장 큰 고유값에 해당하는 고유벡터를 팩터로 사용
v = eigenvecs[:, -r_i:]  # 상위 r_i개 고유벡터
```

## 5. 해석 방법

### 글로벌 팩터 기준 시각화
- **멀티 시계열 플롯**: 각 시계열은 글로벌 팩터와 로컬 팩터의 선형 결합
- **해석**: 
  - 시계열 = 글로벌 팩터의 기여도 + 해당 블록 로컬 팩터의 기여도 + 특이 충격
  - 각 시계열의 변동을 글로벌/로컬/특이로 분해 가능

### 블록 요인들을 쌓은 벡터
- **구조**: `F_t = [F_global, F_consumption, F_invest, F_extern]'`
- **해석의 어려움**: 
  - 각 팩터는 독립적으로 추정되지만, 경제적으로는 상관관계가 있을 수 있음
  - 하지만 모델은 블록 간 독립을 가정하므로, 해석 시 주의 필요

## 6. 금융에서의 활용

### 주요 응용
1. **시장 위험 분석**: 글로벌 팩터 = 전체 시장 위험, 로컬 팩터 = 섹터별 위험
2. **포트폴리오 구성**: 팩터 노출도(factor exposure) 기반 자산 배분
3. **리스크 분해**: 포트폴리오 리스크를 글로벌/로컬/특이 리스크로 분해
4. **스트레스 테스트**: 각 팩터의 충격이 포트폴리오에 미치는 영향 분석

### 예시
- **글로벌 팩터**: 전체 주식 시장의 공통 변동 (시장 리스크)
- **로컬 팩터**: 
  - 금융 섹터 팩터
  - 기술 섹터 팩터
  - 에너지 섹터 팩터

## 7. 코드 구현 요약

### 초기화 과정 (`init_conditions`)
1. 각 블록마다:
   - 해당 블록에 로드되는 시계열 추출
   - Residual covariance 계산
   - Eigendecomposition으로 팩터 추출
   - 로딩 행렬 C 업데이트
   - 팩터 공분산 Q 업데이트

2. 블록 대각 구조:
   - Q 행렬은 블록 대각 행렬
   - 각 블록의 팩터는 독립적으로 추정

### EM 알고리즘
- E-step: Kalman filter/smoother로 팩터 추정
- M-step: 로딩 행렬 C와 공분산 행렬 Q, R 업데이트
- 블록 구조 유지하면서 최적화

## 8. 핵심 정리

1. **글로벌 팩터는 벡터**이고, 로컬 팩터들을 concat한 것이 아님
2. **Block DFM은 요인 회전을 하지 않음** (블록 구조가 이미 해석 제공)
3. **요인간 공분산은 독립 가정** (블록 대각 구조)
4. **계층형 구조**: 글로벌 → 로컬 → 특이 충격
5. **분산-공분산 분해**: 원래 값과 공분산 모두 팩터로 분해
6. **고유분해 사용**: 각 블록의 residual covariance를 eigendecomposition하여 팩터 추출
7. **해석**: 각 시계열의 변동을 글로벌/로컬/특이로 분해하여 분석
